Отчет о лабораторной работе: Использование Git и GitHub
Цель работы
Целью данной лабораторной работы является изучение основ работы с системой контроля версий Git и платформой GitHub. В ходе работы мы научимся инициализировать репозиторий, добавлять файлы, коммитить изменения и отправлять их на удалённый репозиторий.

Оборудование и программное обеспечение
Компьютер с установленной операционной системой (Windows, macOS или Linux).
Установленный Git (версия 2.x или выше).
Учетная запись на GitHub.
Ход работы
Шаг 1: Установка Git
Скачал и установил Git с официального сайта git-scm.com.
Проверил установку, выполнив команду в терминале:
bash

Копировать код
git --version
Убедился, что версия Git отображается корректно.
Шаг 2: Настройка имени пользователя и электронной почты
Открыл терминал (или Git Bash).
Настроил имя пользователя и адрес электронной почты для Git:
bash

Копировать код
git config --global user.name "Mika"
git config --global user.email "ваш_email@example.com"
Проверил настройки:
bash

Копировать код
git config --global --list
Шаг 3: Создание удалённого репозитория на GitHub
Перешел на сайт GitHub и вошел в свою учетную запись.
Нажал на кнопку "+" в правом верхнем углу и выбрал "New repository".
Ввел имя репозитория, добавил описание и выбрал тип репозитория (публичный или приватный).
Нажал на кнопку "Create repository".
Шаг 4: Инициализация локального репозитория
В терминале перешел в директорию, где будет находиться проект:
bash

Копировать код
cd путь/к/вашему/проекту
Инициализировал новый репозиторий Git:
bash

Копировать код
git init
Шаг 5: Создание файла
Создал новый файл с именем example.txt:
bash

Копировать код
touch example.txt
Открыл файл для редактирования и добавил текст:
bash

Копировать код
nano example.txt
После редактирования сохранил изменения и закрыл редактор.
Шаг 6: Добавление файла в индекс
Добавил файл в индекс:
bash

Копировать код
git add example.txt
Шаг 7: Создание коммита
Создал коммит с описанием изменений:
bash

Копировать код
git commit -m "Добавлен файл example.txt"
Шаг 8: Связывание локального репозитория с удалённым
Добавил удалённый репозиторий на GitHub как origin:
bash

Копировать код
git remote add origin https://github.com/USERNAME/REPOSITORY_NAME.git
Шаг 9: Отправка изменений в удалённый репозиторий
Отправил изменения в удалённый репозиторий:
bash

Копировать код
git push -u origin master
(или main, если основная ветка называется так).
Шаг 10: Проверка на GitHub
Перешел на страницу репозитория на GitHub и убедился, что файл example.txt был успешно добавлен.
Заключение
В ходе лабораторной работы я изучил основные команды и принципы работы с Git и GitHub. Научился инициализировать репозиторий, добавлять файлы, коммитить изменения и отправлять их на удалённый репозиторий. Эти навыки являются основополагающими для работы в команде и управления проектами в области разработки программного обеспечения.
